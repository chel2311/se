🔎効率的な検索テクニック

基本的な検索（Ctrl+F）

検索ダイアログの使いこなし方：

  • 大文字小文字の区別：必要に応じてチェック
  • 単語単位：部分一致を避けたい場合にチェック
  • 検索方向：上方向/下方向を選択可能
' 効率的な検索のコツ ' × 悪い例："Click" → 大量にヒット ' ○ 良い例："cmdSave_Click" → 具体的な関数名 ' ◎ 最良："Private Sub cmdSave_Click" → 宣言部分を特定

プロジェクト全体検索

Ctrl+Fダイアログで「現在のプロジェクト」を選択：

手順：
  1. Ctrl+F で検索ダイアログを開く
  2. 検索する文字列を入力
  3. 「検索対象」で「現在のプロジェクト」を選択
  4. 「すべて検索」をクリック
  5. 検索結果ウィンドウから該当箇所へジャンプ

プロジェクト全体検索が有効な場面：

  • グローバル変数の使用箇所をすべて確認
  • 特定の関数の呼び出し元をすべて特定
  • 定数の使用状況を把握
  • 特定のAPIの使用箇所を確認

🔍イミディエイトウィンドウでデバッグ効率UP

イミディエイトウィンドウとは？ VB6.0 IDEに標準搭載されている、コードを1行ずつ即座に実行できる強力なデバッグツールです。 Ctrl + G で瞬時に開けます！

ケース別の推奨アプローチ

バグ調査の場合

  1. エラーが発生する画面・操作を特定
  2. 該当するイベントハンドラにブレークポイント設定
  3. ステップ実行で変数の値を監視
  4. エラー発生箇所を特定したら、関連する関数をShift+F2で追跡
  5. 修正後、同じ手順で動作確認

機能追加の場合

  1. 類似機能の実装を検索（Ctrl+F → プロジェクト全体）
  2. 参考になるコードをF2（オブジェクトブラウザ）で構造確認
  3. 新規関数の配置場所を決定
  4. 既存コードとの連携部分をShift+F2で確認
  5. テスト時はデバッグ機能で動作検証

リファクタリングの場合

  1. MZ-Toolsで対象関数の全使用箇所を確認
  2. 影響範囲をプロジェクト全体検索で特定
  3. Gitで現在の状態をコミット（バックアップ）
  4. 段階的に修正し、都度デバッグ実行で確認
  5. 全体テスト実施
